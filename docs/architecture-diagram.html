UploadPhase components/UploadPhase.tsx

Responsibility
Drag-and-drop or click-to-browse file upload for .tar.gz support bundles. Validates file type and size client-side before sending to the backend.
Interface
Accepts onUploadComplete(sessionId, manifest, signals) callback. Sends POST /api/upload as multipart/form-data.
Key behaviors
Animated drop zone with visual feedback. Progress indication during upload. Error toast on invalid file type or server rejection.

ReportPhase components/ReportPhase.tsx

Responsibility
Displays a live progress stepper during LLM analysis, then renders the structured diagnostic report: executive summary, severity-sorted findings with root causes and remediations, event timeline, and source citations.
Interface
Consumes SSE stream from GET /api/analyze/:id. Fires onReportComplete(report) when the DiagnosticReport is received.
Key behaviors
Real-time progress hints during analysis. Severity filter (all / critical+warning / critical). Copy-to-clipboard on remediations. Clickable source citations open FileViewer. Markdown export.
Depends on
useSSE Timeline FileViewer

ChatPhase components/ChatPhase.tsx

Responsibility
Interactive follow-up chat for investigating findings. Sends user questions and renders streaming LLM responses with tool-use indicators when the AI inspects bundle files.
Interface
Sends POST /api/chat/:id with message history. Consumes SSE response stream. Tool-use events trigger FileViewer panel.
Key behaviors
Streaming message display with typing animation. Tool-use badges showing which files the AI is reading. Chat history maintained in component state (max 40 messages). Session-scoped — resets on new upload.
Depends on

useSSE hooks/useSSE.ts

Responsibility
Custom React hook that manages Server-Sent Event connections. Parses the SSE protocol, dispatches typed events (chunk, report, error, warning, done) to callbacks, and handles connection lifecycle.
Interface
Returns { isStreaming, error, startStream, stopStream }. Accepts onChunk, onEvent, onError, onWarning, onDone callbacks.
Key behaviors
Uses fetch with ReadableStream reader (not native EventSource) for better control. Callbacks stored in refs to avoid stale closures. AbortController for cancellation. Automatic cleanup on unmount.

API Layer api/routes.py

Responsibility
FastAPI router defining all HTTP endpoints. Handles file upload via multipart, triggers analysis and chat as SSE streams, and manages session lifecycle (create on upload, delete on request).
Endpoints
POST /api/upload → parse, classify, create session
GET /api/analyze/:id → SSE stream (chunk, report, error, done)
POST /api/chat/:id → SSE stream with tool-use
DELETE /api/sessions/:id → cleanup
GET /api/health → liveness check
Key behaviors
LLM provider auto-fallback on failure. Markdown fence stripping on LLM output. Signal type sanitization for schema compliance. Concurrency guard prevents duplicate analysis.
Depends on
Bundle Parser Signal Classifier Context Assembler LLM Provider Session Store LLM Logger

Bundle Parser bundle/parser.py

Responsibility
Accept uploaded .tar.gz files, validate format (magic bytes check) and size (500 MB limit), extract all files in memory, and produce a BundleManifest listing every file with its path and size.
Interface
parse_bundle(file: UploadFile) → tuple[BundleManifest, dict[str, bytes]]
Security
Validates tar.gz magic bytes before extraction. Prevents path traversal attacks (rejects entries with .. or absolute paths). All content held in memory — nothing written to disk (GR-6).
Depends on
Python tarfile Python io

Signal Classifier bundle/classifier.py

Responsibility
Classify each file in the bundle manifest into one of six signal types based on path patterns matching Troubleshoot directory conventions.
Signal types
pod_logs, cluster_info, resource_definitions, events, node_status, other
Design
Pure function, no side effects. Pattern-based regex matching on file paths only — no content inspection. Unknown paths fall through to other. Supports arbitrary bundle structures (GR-12).
Depends on
None (pure logic)

Context Assembler analysis/context.py

Responsibility
Assemble classified bundle content into an LLM-ready prompt. Applies priority-based truncation to fit within a 100K token budget (~400K characters).
Truncation priority
Events > Pod Logs > Cluster Info > Resource Definitions > Node Status. Within each type, most recent content is preserved. Truncated sections are annotated in the output.
Interface
assemble_context(classified, files, token_budget) → AnalysisContext
Depends on
Signal Classifier output Extracted file contents

LLM Provider llm/provider.py

Responsibility
Abstract interface for LLM API calls with two implementations: Anthropic Claude (primary) and OpenAI GPT (fallback). Both support streaming responses and tool-use for chat.
Interface
analyze(context) → AsyncIterator[str] — streaming analysis
chat(messages, tools, tool_handler) → AsyncIterator[str] — streaming chat with tool calls
Fallback strategy
Factory reads LLM_PROVIDER env var. If primary fails (auth, rate limit, connection), API layer automatically retries with the other provider. Both providers track input/output tokens for logging.
Depends on
anthropic SDK openai SDK Prompt templates

Session Store sessions/store.py

Responsibility
In-memory storage of session objects. Each session holds the bundle manifest, extracted files, classified signals, diagnostic report, and chat history. No persistence — server restart clears everything.
Interface
create(manifest, files, classified) → Session
get(session_id) → Session
delete(session_id)
GR-6 compliance
All data in-memory only. Delete clears all references. No disk writes, no external storage. Bundle content never leaves the process boundary except to LLM APIs.
Depends on
None (Python dict)

LLM Logger logging/llm_logger.py

Responsibility
Emit structured JSON log entries to stdout for every LLM API call. Provides observability into model usage, costs, and performance without exposing sensitive bundle content.
Log fields
timestamp, session_id, call_type, provider, model, input_tokens, output_tokens, latency_ms, status
GR-6 compliance
No bundle content, user queries, or LLM responses are included in log entries. Only metadata for operational monitoring.

Python json Context manager protocol

Anthropic Claude llm/anthropic_provider.py

Responsibility
Primary LLM provider using the Anthropic Messages API. Streams analysis responses as text chunks and supports multi-turn tool-use in chat via the native Anthropic tool calling protocol.
Model
Configurable via ANTHROPIC_MODEL env var. Uses messages.stream() for token-by-token delivery.
Error handling
Maps SDK exceptions (AuthenticationError, RateLimitError, APIConnectionError) to unified LLMError type. Tracks input/output tokens from response usage metadata.
Depends on
anthropic Python SDK ANTHROPIC_API_KEY

OpenAI GPT llm/openai_provider.py

Responsibility
Fallback LLM provider using the OpenAI Chat Completions API. Streams analysis and chat responses. Handles multi-round tool-use with function calling, accumulating tool call arguments across stream chunks.
Model
Configurable via OPENAI_MODEL env var (default: gpt-4o). Uses chat.completions.create(stream=True) with usage tracking.
Tool-use protocol
Accumulates tool call deltas by index across stream chunks. Executes tools via handler callback, appends results as tool messages, loops up to MAX_TOOL_ROUNDS.
Depends on
openai Python SDK OPENAI_API_KEY
anthropic SDK openai SDK Auto-fallback on failure
